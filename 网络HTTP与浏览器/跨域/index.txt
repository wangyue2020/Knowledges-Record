// 什么是跨域？
// 浏览器去请求资源时，出现协议、域名、端口任一不同，都属于跨域。它是由浏览器的同源策略造成的，是浏览器对JS实施的安全限制。

// 解决方式：
// 1.最流行的跨域解决方案：跨域资源共享（cors）。通常是后端通过设置额外的HTTP头来告诉浏览器，让运行在一个origin上的web应用
//   被准许访问来自不同源服务器上的指定资源。
// 例：res.header('Access-Control-Allow-Origin','*') res.header('Access-Control-Allow-Headers','*')
// 优：比较简便，支持多种请求方式。 劣：低版本IE不兼容。

// 2.最简单的跨域解决方案：反向代理接口跨域（nginx）。原理：同源策略是浏览器的安全策略，不是HTTP协议的一部分。服务器之间不存
// 在跨域问题。

// 3.最经典的跨域解决方案：JSONP。原理：<script/><link/><img/>这些标签没有同源限制。
// 优：利用原生标签的特性，兼容性好。 劣：只支持get方式，并且需要前后端约定。
